    La première ligne du résultat de la commande host indique l'adresse IP de la machine associée au nom de domaine spécifié.

    La deuxième ligne indique la machine qui héberge le serveur de mail du nom de domaine spécifié, l'indice de priorité est aussi spécifié en cas de plusieurs serveurs.

  3. A votre avis, pourquoi utilise-t-on des noms pour les machines plutôt que des adresses IP ?

    L'ajout de nom de domaines à plusieurs intérêts :

    • Cela est plus facile à utiliser pour les humains, retenir et manipuler un nom de domaine est plus simple qu'une suite de chiffre.

    • Cela permet d'abstraire les potentiels modifications de l'infrastructure du serveur. Par exemple si on change l'adresse IP de la machine hébergeant le site et qu'on met bien à jour la modification pour le DNS, l'utilisateur ne s'en rendra même pas compte.

    • Cela permet à l'infrastructure du serveur d'associer plusieurs adresse IP derrière un nom de domaine afin d'utiliser un serveur plus proche géographiquement de l'utilisateur ou pour anticiper des pannes d'un des serveurs.

Exercice 3 : Service HTTP

  1. On cherche à savoir à quoi correspond le fichier /etc/services.

    Le fichier /etc/services contient la liste des services du réseau Internet. Plus précisément, pour chaque service on retrouve le port de communication associé sur la machine, le protocole de transport (couche réseau du modèle OSI) associé et les alias pouvant exister.

  2. Utilisez la commande grep sur le fichier /etc/services pour déterminer le numéro du port TCP correspondant au service http.

    Pour trouver le port TCP pour le service on peut utiliser la commande suivante :

    $ cat /etc/services | egrep '^http\s.*tcp'
    http 80/tcp

    C'est le port 80.

  3. Quel est le header de la réponse ?

    En essayant de se connecter avec netcat au serveur web de www.example.com, voici l'entête de réponse que l'on obtient :

    HTTP/1.0 404 Not Found
    Content-Type: text/html
    Date: Tue, 08 Oct 2024 15:24:35 GMT
    Server: ECAcc (dcd/7D26)
    Content-Length: 345
    Connection: close

  4. A quoi correspondent les 3 valeurs sur la première ligne de l'entête ?

    Les 3 valeurs sur la première ligne renseigne la version du protocole http qui est utilisé (1.0), le code du status de la requête http (404) et le message correspond au code du status http (Not Found).

  5. A quoi correspond la valeur de Content-Type ?

    La valeur Content-Type indique le format de la réponse, ici c'est de l'html.

  6. A quoi correspond la valeur de Content-Length ?

    La valeur Content-Length indique la taille du contenu de la réponse, ici 345 caractères.

  7. En ajoutant un host dans la requête, l'entête de la réponse est :

    HTTP/1.1 200 OK
    Accept-Ranges: bytes
    Age: 420562
    Cache-Control: max-age=604800
    Content-Type: text/html; charset=UTF-8
    Date: Tue, 08 Oct 2024 15:32:17 GMT
    Etag: "3147526947+gzip"
    Expires: Tue, 15 Oct 2024 15:32:17 GMT
    Last-Modified: Thu, 17 Oct 2019 07:18:26 GMT
    Server: ECAcc (dcd/7D65)
    Vary: Accept-Encoding
    X-Cache: HIT
    Content-Length: 1256

    La version d'http est la même, le code du status de la requête est cette fois 200 et correspond au status OK.

    Le format de la réponse est aussi en html, mais cette fois l'encodage des caractères est précisé : UTF-8.

    La taille du contenu est de 1256 caractères.

  8. Pourquoi dans la version 1.1 du protocole HTTP, on doit fournir le Host ?

    Spécifier le host permet aux serveurs d'héberger plusieurs sites web, la version 1.1 l'a simplement rendu obligatoire pour s'accorder avec l'expension d'internet. Il est aussi possible de spécifier l'host avec la version 0.9 mais ce n'est pas obligatoire.

  9. À quoi correspond la valeur de Location ?

    L'entête pour la requête vers www.forax.org est la suivante :

    HTTP/1.1 302 Found
    Date: Tue, 08 Oct 2024 15:47:39 GMT
    Content-Type: text/html
    Transfer-Encoding: chunked
    Connection: keep-alive
    Location: http://www-igm.univ-mlv.fr/~forax
    Vary: Accept-Language
    CF-Cache-Status: DYNAMIC

    On constate que le code de status de la requête n'est pas 200 mais 302, cela signifie que l'on a accédé à la bonne ressource mais par une redirection.

    La valeur correspondant à Location indique où est-ce qu'on a été redirigé.
